# Churn_Preventor

Проект для анализа пользовательских сессий и рекомендации следующего действия, которое может повысить вероятность конверсии и предотваритить уход пользователя.

Например, если пользователь просматривает сайт, но есть риск, что он уйдёт без целевого действия, система анализирует его поведение и предлагает действие, которое может помочь довести пользователя до конверсии.

## Что делает проект

Проект решает задачу **next-best-action recommendation** — выбора следующего лучшего действия для пользователя на основе его поведения в текущей сессии.

В проекте реализован полный ML-пайплайн:

1. Подготовка пользовательских сессий
2. Генерация признаков по действиям пользователя
3. Оценка состояния сессии с помощью HMM
4. Рекомендация действия с помощью Offline Reinforcement Learning
5. FastAPI-сервис для инференса
6. Простой web-интерфейс
7. Запуск через Docker

## Основная идея

Пользовательская сессия представляется как последовательность действий:

```text
Главная страница → Каталог → Карточка товара → Поиск → ...
```

На основе этой последовательности система пытается понять:

* насколько пользователь близок к конверсии;
* какие действия он уже совершил;
* какое действие лучше порекомендовать следующим.

## Используемые подходы

### HMM

HMM используется для оценки скрытого состояния пользовательской сессии.

Идея в том, что по видимым действиям пользователя можно примерно определить его текущее поведенческое состояние: например, пользователь заинтересован, сомневается или близок к уходу.

### Offline Reinforcement Learning

Offline RL используется для выбора следующего действия.

Модель обучается не на взаимодействии с реальными пользователями, а на исторических данных. Это позволяет построить рекомендательную политику без проведения онлайн-экспериментов.

## Результат

Мы не имели доступа к ресурсу, поэтому проводили offline-оценку, а не полноценный A/B-тест на реальных пользователях.
В рамках offline-оценки система показала улучшение примерно на **13%** по proxy-метрике вероятности конверсии.

## Как запустить

### 1. Клонировать репозиторий

```bash
git clone https://github.com/DrStrLove/Churn_Preventor.git
cd Churn_Preventor
```

### 2. Запустить сервис через Docker

```bash
cd web
docker build -t churn-preventor .
docker run -p 8000:8000 churn-preventor
```

После запуска сервис будет доступен по адресу:

```text
http://localhost:8000
```

Документация FastAPI доступна по адресу:

```text
http://localhost:8000/docs
```

## Данные

В проекте использовались исторические данные пользовательских сессий.  Исходные данные включали ид пользователей, временные метки событий и названия событый, url переходов и некоторые мусорные данные.

Данные предобрабатывались и приводились к следующему формату:

| Поле       | Описание                              |
| ---------- | ------------------------------------- |
| session_id | Идентификатор пользовательской сессии |
| user_id    | Идентификатор пользователя            |
| timestamp  | Время действия                        |
| action     | Действие пользователя                 |
| target     | Признак целевого действия / конверсии |

Код проекта, notebooks и web-сервис доступны в репозитории, но исходные данные не включены.
